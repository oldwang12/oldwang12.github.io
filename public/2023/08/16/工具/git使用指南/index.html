4. git reset

放弃所有更改并回到上一次提交的状态：

这将删除所有的未提交更改，将HEAD指向父提交，并将工作区和暂存区恢复到上一次提交的状态。

保留更改但将其从暂存区中移除：

这将将所有已暂存的更改重置，但保留在工作区中，这样你就可以重新提交或进行进一步的更改。

请注意，git reset 是一个潜在的危险操作，因为它会从版本历史中移除提交。在执行这些命令之前，请确保你理解这些操作的副作用，并且在对你的代码产生重大影响之前，最好进行备份或咨询团队中的其他成员。
